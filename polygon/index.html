Polygon Carbon Footprint Methodology

The emissions due to holding and transacting Polygon (MATIC) are calculated using the methodology outlined here, with the following modifications, data sources, and assumptions.

Electricity Consumption

Polygon uses 100 nodes to secure the network (this number was 90 until 2020-04-30[^1]). To account for temporary fluctuations, since 2022-07-08 we use realtime data on the number of validators from the Polygon staking page.
These nodes run two processes, “validator” and “sentry”, each drawing up to 500W of electricity[^1]. Although other estimates[^2] modify this electricity consumption using a load factor of between 0.75 and 0.94 to allow for the nodes performing other general purpose computation tasks, we do not believe this assumption is justified. So, we use the maximum 1,000W electricity consumption per node to calculate the total electricity consumed by the network.

Carbon emissions

Given this geographic breakdown, we use emissions factors for grid-purchased electricity in the various regions to calculate the total carbon emissions due to the electricity consumption of the Polygon network.
The emissions factors for grid-purchased electricity in the respective regions come from various national and multi-national government sources including the US EPA, and the European AIB, as well as climate NGOs such as the Institute for Global Environmental Strategies, Greenhouse Gas Protocol, and Climate Transparency.

Checkpointing and Bridging on Ethereum

The Polygon network writes a checkpoint transaction to the Ethereum mainnet periodically (roughly every 45 minutes). Users also bridge various cryptocurrency tokens between Polygon and the Ethereum mainnet. So, Polygon is responsible for a portion of the Ethereum mainnet carbon emissions.
The checkpoint and bridge contracts do not hold any significant value for any length of time, so we only consider the share of the Ethereum mainnet carbon emissions due to transactions. We do not factor in any emissions due from holding value on the Ethereum mainnet.

Blockchain Metrics

  • Ethereum mainnet transaction fees are retrieved using Etherscan, filtering for transactions relating to the Polygon checkpoint smart contract address 0x86E4Dc95c7FBdBf52e33D563BbDB00823894C287, and the main Polygon bridge smart contract address 0xa0c68c638235ee32657e8f720a23cec1bfc77c77
  • Daily circulating supply of MATIC, transaction count and transaction fees are aggregated from Polygon on-chain data hosted on BigQuery public datasets
  • Polygon staking rewards are retrieved from the Polygon staking page [^3]

Assumptions and limitations

  • We are assuming that 1,000W is a realistic estimate for the electricity consumption of a Polygon node.
  • This methodology assumes that all Polygon node operators use electricity purchased from the national grid, and therefore has the same emissions profile as the grid. This is very likely to be true, since Proof of Stake blockchain nodes do not require unusual amounts of power, so electricity price is unlikely to represent a significant cost to node operators.
  • This methodology does not yet include embedded carbon emissions from the manufacture and disposal of the computers used as Polygon nodes.
