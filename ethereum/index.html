Ethereum Carbon Footprint Methodology

The emissions due to holding and transacting Ethereum (ETH) are calculated using the methodology outlined here, with the following modifications, data sources, and assumptions.
Since The Merge on 2022-09-15 Ethereum is a Proof of Stake (PoS) cryptocurrency. Prior to The Merge it used Proof of Work (PoW) mining, which requires approximately 2,000 times as much energy. For our pre-Merge methodology, please see the pre-merge version of this document.

Electricity Consumption

The Ethereum documentation gives the following hardware validator requirements:
  • Minimum of 6667+ CPU Benchmark Score
  • 16 GB RAM (but 32 GB is recommended)
  • 2 TB SSD
  • Display screen and keyboard (unless running DAppNode, or ssh/headless setup)
Using OuterVision’s PSU calculator we estimate 242W as the power consumption.
Other constants used in the calculation for energy consumption:
  • Data center power usage effectiveness - we have elected to use a figure of 1.1 for this which matches CBECI’s best guess estimate.
  • Power Supply Efficiency - We have chosen a figure of 90% PSU from another studies’ approach to calculating emissions for Ethereum.

Geographic distribution

Geographic distribution is fetched from nodewatch.io. This is from a list of recommended sources for statistics on Nodes from Ethereum and also has the highest number of nodes in the statistics. We take the top 50 countries which have the highest number of nodes - as of 2022-09-27 this roughly equates to 99.3% of all nodes.
This is so that we do not have to continuously update emission factors if a node is added to a country for which we have no grid-purchased electricity emission factor data.
The distribution we can retrieve daily. However, for the days for which we have no distribution data available we backfill, i.e. use today’s distribution for yesterday (if yesterday is missing data).

Blockchain Metrics

  • Validator count from etherscan validators page
  • Staking rewards from beaconcha.in
  • Transaction count, circulating supply and transaction fees are calculated from Ethereum blockchain data in BigQuery public dataset

Assumptions and limitations

  • This methodology does not yet include embedded carbon emissions from the manufacture and disposal of the computers used as validator nodes.
