Eunice Data API

API service for eunice-data-api.
Implemented using python FastAPI framework.
Authentication and authorisation are handled by AWS Cognito user pool. See terraform scripts for more information.

Auth flow (subject to change)

At the moment, user sign-up and sign-in are not allowed. A new user has to be added by an admin via aws cli. Each user will be assign a random password for the sake of setup but users are not allowed to see or change the password. We don’t store the password either.
We generate a refresh token and send the refresh token together with client_id to a user. The user can then use refresh token and client id to generate short lived access token. (Later on we should allow a registered user to use our webapp to create and revoke their refresh token by themselves)
If a user refresh token expired, they will have to contact us to generate a new one for them.
If we need to generate an refresh token key for a user, reset the password every time.
This setup is for trial users and may change when we are ready to deploy for full production usage.

Add new user or update existing one

We include a script cognito_user.sh to ease the user setup for now. This script can be to add a new user the first time and can also be used to reset user’s refresh token. The output from the script is a json file containing all information (client id and refresh token etc) for generating user access token by themselves. This JSON file should be sent to a trial user securely.
# Navigate to root folder of the repo

./scripts/cognito_user.sh  -u ‘<username>’ -s ‘<scopes>’ -r ‘<aws region>’


# Example
./scripts/cognito_user.sh  -u ‘test-user’ -s ‘user:estimate user:metrics user:emissions’ -r ‘eu-west-2’

Remove a user

aws cognito-idp admin-delete-user \
    --user-pool-id $(terraform/service output -raw user_pool_id) \
    --username '<username>'

Get short-lived user access token to access the API

AUTH_URL=$(terraform -chdir=./terraform/service output -raw oauth_url)
CLIENT_ID=$(terraform -chdir=./terraform/service output -raw user_pool_client_id)
REFRESH_TOKEN=<Refresh token of a user>

curl --location --request POST 

“
CLIENT_ID&refresh_token=$REFRESH_TOKEN” 

–header ‘Content-Type: application/x-www-form-urlencoded’
